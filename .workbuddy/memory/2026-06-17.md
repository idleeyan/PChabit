### 修复的 14 条陷阱
本次升级过程中发现并修复了 14 条已知陷阱：

1. **陷阱 11**：SQLitePCLRaw 原生库 `e_sqlite3.dll` 缺失导致所有页面功能失效
   - 根因：`dotnet publish` 未指定 `-r win-x64`
   - 修复：在 csproj 中添加 `<RuntimeIdentifier>win-x64</RuntimeIdentifier>`

2. **陷阱 12**：DashboardViewModel 缺少 DailySummaries 表创建 + 分类筛选硬编码中文匹配
   - 根因：`DatabaseInitializer.MigrateAnalysisTablesAsync()` 遗漏了 DailySummaries 表
   - 修复：补建 DailySummaries 表 + 分类筛选改为通过 WebsiteDomainMappings 表查询

3. **陷阱 13**：SankeyView 数据丢失 — WebView2 初始化阻塞数据加载
   - 根因：数据加载仅在 `WebView2.NavigationCompleted` 事件中调用
   - 修复：数据加载与 WebView2 初始化并发进行

4. **陷阱 14**：退出程序时 SQLite WAL checkpoint 导致系统卡顿
   - 根因：`ServiceProvider.Dispose()` 触发 WAL checkpoint 产生大量磁盘 I/O
   - 修复：跳过 `ServiceProvider.Dispose()`，直接 `ForceTerminate()`

### WMC9999 编译错误修复
**根因链**：
1. SankeyView.xaml 中 `{x:Bind ViewModel.StartDate}` / `{x:Bind ViewModel.EndDate}` / `{x:Bind ViewModel.TopN}` 引用了 SankeyViewModel 中不存在的属性
2. XAML 编译器检测到绑定错误后，尝试调用 `ResourceManager.GetString()` 本地化错误消息为 zh-CN
3. WinAppSDK 2.2.1 的 XamlCompiler 在 zh-CN 系统上缺少中文卫星程序集资源
4. 错误报告机制自身崩溃 → WMC9999 吞没真正的绑定错误

**修复**：为 SankeyViewModel 添加 StartDate（DateTimeOffset?）、EndDate（DateTimeOffset?）、TopN（int）属性

## v3.0.1 周期卡顿修复

### 问题
用户报告程序每隔几分钟卡顿，导致鼠标移动困难。

### 根因分析（2 个因素叠加放大）
**因素 1**：Serilog Debug 级别同步日志在钩子线程中阻塞
- `App.xaml.cs` 最低日志级别设为 `Debug`，File Sink 为同步写入
- `MouseMonitor.HookCallback` 和 `KeyboardMonitor.HookCallback` 中每条事件都调用 `Log.Debug()`
- `DataCollectionService` 的事件处理器中也调用 `Log.Debug()`
- Windows 低级钩子回调在安装钩子的线程执行，同步文件 I/O 阻塞该线程 → 全局输入消息排队

**因素 2**：`wal_autocheckpoint=200` 阈值过低
- 默认 200 页（~800KB），正常使用约 8 分钟触发一次 checkpoint
- Checkpoint 产生密集磁盘 I/O，与同步日志写入叠加 → 大幅延长阻塞时间

### 修复
1. **Serilog 配置**：`MinimumLevel.Information()` + `WriteTo.Async()`
2. **钩子回调**：移除 `MouseMonitor.cs:87` 和 `KeyboardMonitor.cs:107-108` 中的 `Log.Debug`
3. **事件处理器**：移除 `DataCollectionService.cs:307` 和 `452` 中的 `Log.Debug`
4. **WAL checkpoint**：`wal_autocheckpoint` 从 200 → 10000
5. **NuGet**：添加 `Serilog.Sinks.Async` 2.* 依赖
6. **版本**：3.0.0 → 3.0.1

### 编译问题
XAML 编译器在命令行环境下崩溃 (`output.json was not created`)，此为 Trap 10 的预存问题。项目之前在 VS 中编译正常，因 obj 缓存中已有预编译 XAML。需通过 Visual Studio 编译。
